    1.3.1 environment

    for a function, there is a function entity and a function name

    intrinsic name: 从函数的功能上来说，执行相同的步骤的函数都有相同的intrinsic name frame name: 从环境来说，只是一个变量名，可以指向函数，数据等等

    function signatures: 如何唯一地标识一个函数，函数名+参数列表

    1.3.2 calling user-defined functions

    apply a user-defined function to some argument 1. bind the argument to the formal parameter of this function to form local frame 2. excute the body of the function in the environment that start with this frame（local frame + global frame，每次遇到一个变量，先到local frame找，然后再到global frame找)

    更加严谨的说，frame形成了一个栈，当我们需要查找一个变量时，从栈顶依次找，直到找到第一个含有这个变量的frame。这也意味着，在同一个程序里，可以多次定义同一个name，但是有不同的值，甚至有不用的类型

        while not close(guess):
