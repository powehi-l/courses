1. 应用层协议原理

[[Lec1 概论#网络边缘|网络应用的体系结构]]

  • CS client/server
    • 服务器：一直运行
      • 固定的IP地址以及约定好的端口号
      • 扩展性(差)
    • 客户端
      • 主动与服务器通信
      • 动态IP地址
  • Peer to peer
  • 混合模式

进程通信

  • 进程：在主机上运行的应用程序
    • 客户端进程：发起通信的进程
    • 服务器进程：等待连接的进程
  • 通信
    • 同一个主机内，操作系统定义
    • 不同主机：交换报文
  • 需要解决的问题
    • 进程标识和寻址问题
    • 传输层-应用层提供的服务是什么(SAP socket, API)
    • 如何使用传输层提供的服务，实现应用进程之间的报文交换

对进程进行编址

  • 进程在发送接受报文的时候需要有唯一标识
    • 主机：IP
    • 采用的协议
    • 端口号：port number
  • HTTP: TCP 80
  • 一个进程，用IP+port表示端节点

传输层提供的服务

需要穿过层间的信息

  • 必须携带的信息
    • 数据
    • 谁传的(IP + TCP(UDP)端口)
    • 传给谁，对方的(IP + TCP(UDP)端口)

层间信息的代表(TCP)

  • API传输报文容易出错
  • 用socket(一个整数)标识通信的双方（类比os句柄fd）
  • TCP socket: 源IP(给下层IP协议使用[[Lec1 概论#数据单元|ICI + SDU = PDU]])，源端口，目标IP，目标端口
  • 本地的标识，其他主机不知道

层间信息代码(UDP)

  • UDP两个进程无需建立连接
  • UDP socket： 本IP，本端口
  • 传输报文时需要提供对方IP，port

TCP 与 UDP socket间的区别

  • UDP socket 只有自己的IP和socket，相当于唯一标识了一个进程，而TCP socket 相当于在进程基础上表示了一对进程，即一个关系
  • UDP 在调用API时需要向下传递目标IP和目标端口号
