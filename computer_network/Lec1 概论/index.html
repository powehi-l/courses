    1. 什么是Internet

    从具体组成角度

    • 网络：图（节点，边）
    • 计算机网络：由计算机组成节点的网络（路由器，交换机）
      • 节点： 路由器，交换机(end system)
      • 边： 接入网链路，主干链路
      • 协议： 两个节点如何交流
    • 网络的网络

    从服务的角度

    • 通过网络连接起来的分布式应用
    • 通信基础设施为应用提供网络传输服务

    2. 网络边缘

    网络结构

    • 网络边缘
      • 主机
      • 应用程序
    • 网络核心
      • 路由器
      • 交换机
    • 网络接入
      • 连接边缘与核心的部分

    各自的功能

    • 网络应用， 有联网的需求
    • 主机，各种协议栈，为应用联网提供服务
    • 网络核心：传输数据到目的地
    • 网络接入: 将网络连接成更大的网络

    网络边缘

    • 端点的主机
    • 边缘交流的模式
      • cs（client-server模式）： 服务器为客户端提供服务
        • 当客户端增大时，性能下降
      • p2p（peer-to-peer模式）：客户端与客户端提供数据，但是仍然有一个服务器进行调j度

    网络应用采用的服务(由网络基础设施提供)

    采用网络设施的面向连接的服务

    • 进程在通信之前，需要各个层面都知道已经建立连接
    • tcp

    采用基础设施的无连接服务

    • 直接进行发送，网络边缘并不知道对方的状态
    • udp

    3. 网络核心

    由路由器组成的网状网络

    网络的类型

    • 电路交换：专有的线路进行通信，例如电话网
    • 分组交换：将需要传输的数据分成一个个分组，然后交由路由器进行传输

    电路交换

    在两个路由器之间将线路分为若干条线，独享的线路 - 特点 - 不共享 - 保证了性能 - 要求建立连接 - 分割线路的类型（网络资源被分为“片”， 每个片要么空闲，要么被独享） - FDM 频分(将频率的一段分为小段) - TDM 时分(将一个周期分为小段) - 波分(将光波根据波长分为多个段) - 传播延迟 - 电路交换不适合计算机之间的通信 - 建立时间长 - 计算机通信由突发性，随时可能发生或者不发生，会浪费“片” - 可靠性不高，每一对交换机一旦出现问题，则会直接失败

    分组交换

    • 通过网络核心时，每一条使用全部带宽
    • 存储-转发模式，以packet为单位，全部存储后再转发（区分于电路交换，即多个通信可共用）

    存储-转发

    • 必须等到完全接受之后才能进行转发

    排队延迟和丢失

    • 如果前面还没传输完，则需要等待
    • 如果交换机缓存已满，则会被丢弃

    网络核心的关键功能

    • 路由：决定分组采用的源到目标的路径--路由算法
    • 转发：将分组从路由器的输入链路转移到输出链路

    统计多路复用

    • 没有固定的复用方式

    分组交换 vs 电路交换

    分组交换允许更多用户同时使用网络 - 适合突发性数据传输 - 资源共享 - 无需呼叫 - 过度使用会导致拥塞 - 相应的机制

    分组交换的类型

    1. 数据报网络
      • 在分组交换机处决定下一跳的目标
      • 问路
      • Internet
    2. 虚电路网络
      • 每个分组带有标签，决定下一跳
      • 呼叫时决定路径并保持不变
      • 路由器维持每个呼叫的状态信息
      • 信令

    6. 分组延时，丢失，吞吐量

    分组延时

    1. 节点处理延时：分组到达之后需要进行处理
    2. 排队延时：需要等待其他的分组处理完成
    3. 传输延时：L/R 从主机输出bit的时间

    4. 传播延时([[Lec1 概论#排队延迟和丢失|处于网络核心中的分组延时]]): 在线路上传播的时间，如果距离近则可以忽略不记

    5. 排队延时

      • R = 链路带宽
      • L = 分组长度
      • a = 分组到达队列的平均速率
      • 流量强度 = \(La/R\)
        • 当流量强度趋近于1的时候，延时趋近于无限大

    分组丢失

    • 缓存队列有限
    • 可能会被上一个节点或源端系统重传，或者不重传

    吞吐量

    在源端与目标端之间传输的速率 - 瞬间，平均 - 短板效应， 在一条链路上，取决于吞吐量小的 - 实际取决于每一条链路除以同时使用的数量的最小值 即 \(min\{R_i/N_i\}\)

    7. 协议层次及服务模型

    协议与服务

    协议层次

    • 将网络的复杂功能分为不同层次的功能，每一层接口为上一层提供接口，无需担心下层的实现
    • 例子，哲学家的交流， 三层（哲学家，翻译，传报人）

    服务： 下层向上层提供的功能

    • 服务访问点(SAP): 区分上层的用户（调用者）
    • 通过原语向上层提供服务，可以理解为API
    • 服务的类型
      • 面向连接的服务（需要都知道已经建立了连接，为后续链接做好准备
      • 无连接的服务：不需要建立连接

    服务与协议的区别

    • 服务是垂直的关系，下层向上层提供服务
    • 协议：对等层实体进行交流

    服务与协议的关系

    • 协议借助下层服务实现
    • 本层实体为上层提供更高级别的服务

    数据单元

    • 术语
      • SAP: service access point
      • IDU: interface data unit
      • SDU: service data unit
      • PDU: protocol data unit
      • ICI: interface control information
    • \(SDU_n + ICI_n = PDU_n = SDU_{n+1}\)

    分层实现的好处

    • 结构清晰
    • 便于维护和升级：分别实现
    • 坏处：效率降低

    互联网协议栈

    • 应用层：网络应用
      • FTP, HTTP, DNS, SMTP
    • 传输层：主机之间的数据传输
      • 进程到进程的区分，将通信由不可分变为可分
      • TCP，UDP
    • 网络层：end-to-end的数据传输
      • IP
    • 链路层：在相邻两点传输以帧为单位的数据
      • point-to-point: PPP, wifi, ethenet
    • 物理层：将电磁波信号与01数据相互转化

    ISO/OSI 参考模型

    • 七层： 应用层，表示层，会话层，传输层，网络层，链路层，物理层
    • 封装与解封装
      • 交换机：链路层，查看网卡物理地址，选择由哪个端口收发
      • 路由器：网络层，查看IP地址，选择由哪个网卡分发

    各层次协议数据单元

    • 应用层：报文(message)
    • 传输层：报文段(segment)
    • 网络层：分组(packet),无连接(datagram)
    • 数据链路层：帧(frame)
    • 物理层：比特(bit)
